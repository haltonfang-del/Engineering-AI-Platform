# Coding Standards

## Principles

- Readability First
- Documentation First
- SOLID
- DRY
- Keep It Simple (KISS)

---

## Naming

| Item | Convention |
|------|------------|
| Class | PascalCase |
| Function | snake_case |
| Variable | snake_case |
| Constant | UPPER_CASE |
| File | snake_case |

---

## Rules

- Keep functions short and focused.
- Avoid duplicated code.
- Avoid magic numbers.
- Prefer configuration over hardcoding.
- Write self-explanatory code.

---

## Documentation

Public modules and APIs should include documentation.

Complex business logic should explain the reason, not the syntax.

---

## Testing

Future implementations should include unit tests for business logic.
